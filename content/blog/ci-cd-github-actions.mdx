## ¿Por qué GitHub Actions?

GitHub Actions permite automatizar flujos de trabajo directamente desde tu repositorio. Es gratuito para repositorios públicos y tiene una generosa capa gratuita para privados.

## Tu Primer Workflow

Crea un archivo `.github/workflows/ci.yml`:

```yaml
name: CI Pipeline

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v4

    - name: Setup Node.js
      uses: actions/setup-node@v4
      with:
        node-version: '20'
        cache: 'npm'

    - name: Install dependencies
      run: npm ci

    - name: Run tests
      run: npm test

    - name: Build
      run: npm run build
```

## Conceptos Clave

### Triggers (on)

Define cuándo se ejecuta el workflow:

- `push` - Cuando hay commits
- `pull_request` - En PRs
- `schedule` - Cron jobs
- `workflow_dispatch` - Manual

### Jobs y Steps

Los **jobs** corren en paralelo por defecto. Cada job tiene múltiples **steps** que corren secuencialmente.

### Secrets

Guarda información sensible de forma segura:

```yaml
- name: Deploy
  env:
    API_KEY: ${{ secrets.API_KEY }}
  run: ./deploy.sh
```

## Ejemplo: Deploy a Producción

```yaml
deploy:
  needs: build
  runs-on: ubuntu-latest
  if: github.ref == 'refs/heads/main'

  steps:
  - name: Deploy to server
    run: |
      echo "Deploying..."
```

## Mejores Prácticas

1. **Cachea dependencias** para builds más rápidos
2. **Usa matrices** para probar múltiples versiones
3. **Separa jobs** para paralelizar tareas
4. **Protege secrets** nunca los expongas en logs
